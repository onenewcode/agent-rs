# Agentic Design Patterns

This document details the advanced LLM agent patterns implemented in `agent-rs`.

## 1. Tool Calling (Refinement Phase)

Instead of regenerating an entire 5000-word document to fix a single typo, our Agent uses **localized refinement**.

- **`EditDocumentTool`**: Allows the model to find specific text and replace it.
- **`WebSearchTool`**: Allows the model to proactively look up missing facts.

In `RefineStep`, we use `rig-core` to build an agent bound to these tools. The agent "thinks" and performs multiple tool calls to solve the issues identified by the evaluator.

## 2. Multi-dimensional LLM-as-a-Judge

The evaluation node (`EvaluateStep`) uses a structured rubric rather than a binary pass/fail.

| Dimension | Description |
| :--- | :--- |
| **Faithfulness** | Accuracy against provided source materials. |
| **Relevance** | Adherence to the user's specific prompt instructions. |
| **Accuracy** | Chinese grammar, technical structure, and logic. |

The `QualityGate` only passes if thresholds are met across all dimensions.

## 3. High-Concurrency Research

We maximize I/O performance by using `tokio::join!` and `JoinSet`.
- All user-provided URLs are fetched in parallel.
- All search queries generated by the Planner are executed in parallel.
- Both fetching and searching overlap.

This reduces the total wall-clock time of the `ResearchStep` by up to 80%.
